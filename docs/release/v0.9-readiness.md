# v0.9 Readiness Checklist

This branch tracks practical readiness checks for the current `0.9.x` runtime package. It is intended for maintainers reviewing package contents, CLI behavior, and documentation before a release handoff.

## Package contents

- Confirm `npm pack --dry-run --json` includes `dist/`, `README.md`, `CHANGELOG.md`, `LICENSE`, and the public CLI entrypoint.
- Confirm generated declaration files match the public TypeScript exports.
- Confirm no fixture files are required at runtime unless they are intentionally included for examples or tests.
- Confirm `package.json` points `main`, `types`, and `bin.machinefi` at built artifacts.

## CLI smoke checks

- Run `npm run test:smoke` from a clean install.
- Run `npx machinefi status --chain solana --fixture` after a local pack when checking fixture mode manually.
- Confirm failure cases return non-zero status and a readable error for missing command arguments.

## Documentation review

- Confirm README install and status examples match the current package name.
- Confirm changelog notes describe the public SDK/CLI surface without implying a GitHub Release artifact.
- Confirm Solana and Robinhood sections describe source-aware receipt evidence and fixture/live-mode boundaries.

## Compatibility notes

- Keep `0.9.x` API changes additive where possible.
- Preserve receipt verification result shapes used by examples and tests.
- Review adapter exports before changing file paths under `dist/adapters/*`.
